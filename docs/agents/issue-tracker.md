# Issue tracker: GitHub

Issues and PRDs for this repository live as GitHub issues. Use the `gh` CLI for issue operations and infer the repository from the `origin` remote.

## Conventions

- Create issues with `gh issue create`.
- Read the issue body, comments, and labels before acting.
- Apply the labels defined in `docs/agents/triage-labels.md`.
- Close an issue only after its acceptance criteria are satisfied.
